Полная инструкция по использованию команд для настройки виртуальных машин

## Файл 01 [[Образование/Колледж/Мдк/Демо/модуль 1/ISP|ISP]]

### Назначение файла

Настройка маршрутизатора ISP, который имитирует провайдера и обеспечивает доступ в интернет для сетей HQ и BR.

---

1. Установка имени хоста

```bash
hostnamectl hostname ISP
```

· hostnamectl – утилита для управления системными настройками, включая имя хоста.
· hostname ISP – устанавливает имя хоста (hostname) на ISP.
· Зачем: Идентификация устройства в сети, используется в логах, запросах DNS и т.д.

```bash
exec bash
```

· exec – заменяет текущий процесс новым (в данном случае – оболочкой bash).
· Зачем: Для применения изменений окружения (например, обновления переменной $HOSTNAME).

---

2. Проверка конфигурации интерфейса

```bash
cat /etc/net/ifaces/enp7s1/options
```

· cat – выводит содержимое файла.
· /etc/net/ifaces/enp7s1/options – файл конфигурации сетевого интерфейса enp7s1.
· Зачем: Убедиться, что интерфейс имеет правильный тип (например, TYPE=eth).

---

3. Создание структуры каталогов для интерфейсов

```bash
mkdir -p /etc/net/ifaces/enp7s{2,3}
```

· mkdir -p – создаёт каталог и все необходимые родительские каталоги (если их нет).
· enp7s{2,3} – шаблон, раскрывается в enp7s2 и enp7s3.
· Зачем: Для хранения конфигурационных файлов каждого интерфейса в отдельном каталоге (стандарт ALT Linux).

```bash
echo 'TYPE=eth' | tee /etc/net/ifaces/enp7s{2,3}/options
```

· echo – выводит текст.
· tee – записывает в файл и одновременно выводит на экран.
· Зачем: Устанавливает тип интерфейса как Ethernet для enp7s2 и enp7s3.

---

4. Настройка IP-адресов

```bash
echo '172.16.1.1/28' > /etc/net/ifaces/enp7s2/ipv4address
echo '172.16.2.1/28' > /etc/net/ifaces/enp7s3/ipv4address
```

· > – перенаправление вывода в файл (перезаписывает).
· /etc/net/ifaces/.../ipv4address – файл, содержащий IP-адрес и маску подсети.
· Зачем: Назначение статических IP-адресов интерфейсам для связи с HQ-RTR и BR-RTR.

---

5. Установка и настройка nftables

```bash
apt-get update && apt-get install nftables -y
```

· apt-get update – обновляет список доступных пакетов.
· apt-get install nftables -y – устанавливает пакет nftables без запроса подтверждения.
· Зачем: Для настройки межсетевого экрана и трансляции адресов (NAT).

---

6. Создание конфигурации nftables для NAT

```bash
cat << EOF > /etc/nftables/nftables.nft
#!/usr/sbin/nft -f
flush ruleset
table ip nat {
 chain postrouting {
 type nat hook postrouting priority srcnat;
 oifname "enp7s1"  masquerade
 }
}
EOF
```

· cat << EOF > файл – записывает многострочный текст (here-doc) в файл.
· #!/usr/sbin/nft -f – shebang, указывает интерпретатор для скрипта.
· flush ruleset – очищает все текущие правила nftables.
· table ip nat – создаёт таблицу NAT для IPv4.
· chain postrouting – цепочка для пост-маршрутизации.
· type nat hook postrouting priority srcnat – определяет тип цепочки и место в обработке пакетов.
· oifname "enp7s1" masquerade – включает маскарадинг (динамический NAT) для исходящего интерфейса enp7s1.
· Зачем: Чтобы внутренние сети (HQ, BR) могли выходить в интернет через ISP.

```bash
cat /etc/nftables/nftables.nft
```

· Зачем: Проверка корректности записанного конфигурационного файла.

```bash
systemctl enable --now nftables
```

· systemctl enable – добавляет службу в автозагрузку.
· --now – сразу запускает службу.
· Зачем: Активация и автозапуск nftables.

---

7. Включение IP-форвардинга

```bash
sed -i 's/net.ipv4.ip_forward = 0/net.ipv4.ip_forward = 1/' /etc/net/sysctl.conf
```

· sed -i – редактирует файл на месте.
· s/старое/новое/ – команда замены.
· Зачем: Включает маршрутизацию IPv4 между сетевыми интерфейсами.

```bash
systemctl restart network
```

· Зачем: Применение изменений в сетевых настройках.

```bash
sysctl net.ipv4.ip_forward
```

· sysctl – выводит или изменяет параметры ядра.
· Зачем: Проверка, что IP-форвардинг включён (должно быть 1).

---

6. Настройка GRE-туннеля

```bash
cat << EOF > /etc/net/ifaces/gre1/options
TYPE=iptun
TUNTYPE=gre
TUNLOCAL=172.16.2.2
TUNREMOTE=172.16.1.2
TUNTTL=64
TUNOPTIONS='ttl 64'
EOF
```

· TYPE=iptun – указывает, что интерфейс туннельный.
· TUNTYPE=gre – тип туннеля GRE.
· TUNLOCAL – локальный IP-адрес для туннеля.
· TUNREMOTE – удалённый IP-адрес (на HQ-RTR).
· TUNTTL – время жизни пакетов в туннеле.
· Зачем: Создание защищённого туннеля между офисами для передачи трафика.

```bash
cat /etc/net/ifaces/gre1/options
```

· Зачем: Проверка конфигурации.

```bash
echo "10.10.10.2/30" > /etc/net/ifaces/gre1/ipv4address
```

· Зачем: Назначение внутреннего IP-адреса туннельному интерфейсу.

```bash
systemctl restart network
```

· Зачем: Применение сетевых настроек.

```bash
ip -br -c a
```

· ip -br -c a – краткий цветной вывод IP-адресов интерфейсов.
· Зачем: Проверка корректности настройки всех интерфейсов.

9. Настройка NAT (nftables)

11. Создание пользователя net_admin

```bash
useradd net_admin
```

· useradd – создаёт нового пользователя.
· Зачем: Создание учётной записи администратора.

```bash
echo "net_admin:P@ssw0rd" | chpasswd
```

· chpasswd – изменяет пароли в пакетном режиме.
· Зачем: Установка пароля для пользователя.

```bash
usermod -aG wheel net_admin
```

· usermod -aG – добавляет пользователя в группу.
· wheel – группа, члены которой могут выполнять sudo.
· Зачем: Предоставление прав администратора.

```bash
echo "WHEEL_USERS ALL=(ALL:ALL) NOPASSWD: ALL" > /etc/sudoers.d/net_admin
```

· /etc/sudoers.d/ – каталог для дополнительных правил sudo.
· Зачем: Разрешение выполнять любые команды через sudo без пароля.

```bash
su -l net_admin
```

· su -l – переключение на пользователя с загрузкой его окружения.
· Зачем: Проверка входа под новым пользователем.

```bash
sudo id
```

· id – выводит информацию о пользователе и группах.
· Зачем: Проверка, что sudo работает без пароля и показывает UID=0 (root).

---

12. Настройка OSPF (FRR)

```bash
sed -i 's/ospfd=no/ospfd=yes/' /etc/frr/daemons
```

· Зачем: Включение демона OSPF в FRR.

```bash
grep ospf /etc/frr/daemons
```

· Зачем: Проверка изменения.

```bash
cat <<'EOF' > /etc/frr/frr.conf
```

· Зачем: Создание конфигурации OSPF.
· interface gre1 – настройка OSPF на туннельном интерфейсе.
· ip ospf authentication – включение аутентификации.
· ip ospf authentication-key P@ssw0rd – установка ключа аутентификации.
· no ip ospf passive – интерфейс активен в OSPF.
· interface enp7s2 – настройка OSPF на интерфейсе к BR-SRV.
· router ospf – вход в режим конфигурации OSPF.
· passive-interface default – все интерфейсы по умолчанию пассивные (не рассылают OSPF hello).
· Зачем: Обмен маршрутами между HQ и BR через туннель.

```bash
systemctl restart network
systemctl enable --now nftables frr
```

· Зачем: Применение настроек и автозапуск служб.

```bash
cat /etc/resolv.conf
```

· Зачем: Проверка DNS-настроек.

```bash
ip r
```

· ip r – вывод таблицы маршрутизации.
· Зачем: Проверка наличия маршрутов, в т.ч. через OSPF.


· vlan{100,200,999} – создаёт каталоги для VLAN-интерфейсов.
· Зачем: Для настройки VLAN 100 (серверы), 200 (клиенты) и 999 (управление).

```bash
echo $'100\n200\n999' | xargs -i bash -c 'echo -e "TYPE=vlan\nHOST=enp7s2\nVID={}" > /etc/net/ifaces/vlan{}/options'
```

```bash
echo '192.168.100.1/27' > /etc/net/ifaces/vlan100/ipv4address
echo '192.168.200.1/28' > /etc/net/ifaces/vlan200/ipv4address
echo '192.168.99.1/29' > /etc/net/ifaces/vlan999/ipv4address
```

· Зачем: Назначение IP-адресов VLAN-интерфейсам (шлюзы для подсетей).

13. Настройка DHCP (dnsmasq)

```bash
sed -i 's/AUTO_LOCAL_RESOLVER=yes/AUTO_LOCAL_RESOLVER=no/' /etc/sysconfig/dnsmasq
```

· Зачем: Отключает локальный резолвер dnsmasq, чтобы не конфликтовать с Bind9.

```bash
grep AUTO_LOCAL_RESOLVER /etc/sysconfig/dnsmasq
```

· Зачем: Проверка изменения.

```bash
cat <<'EOF' > /etc/dnsmasq.conf
port=0
interface=vlan200
listen-address=192.168.200.1
dhcp-authoritative
dhcp-range=interface:vlan200,192.168.200.2,192.168.200.2,255.255.255.240,6h
dhcp-option=3,192.168.200.1
dhcp-option=6,192.168.100.2
leasefile-ro
EOF
```

· port=0 – отключает DNS-сервер dnsmasq (только DHCP).
· interface=vlan200 – слушать на VLAN 200.
· listen-address=192.168.200.1 – адрес интерфейса для DHCP.
· dhcp-authoritative – сервер является авторитетным для этой подсети.
· dhcp-range – диапазон адресов: только один адрес 192.168.200.2 для HQ-CLI.
· dhcp-option=3 – опция шлюза по умолчанию.
· dhcp-option=6 – опция DNS-сервера.
· leasefile-ro – файл аренды только для чтения (для безопасности).
· Зачем: Автоматическая выдача IP-адреса HQ-CLI с правильным шлюзом и DNS.

```bash
systemctl enable --now frr dnsmasq
```

· Зачем: Запуск служб маршрутизации и DHCP.

```bash
ss -lun | grep 67
```

· ss -lun – показывает слушающие UDP-сокеты.
· 67 – порт DHCP-сервера.
· Зачем: Проверка, что DHCP-сервер запущен.


· ip r | grep ospf – фильтрует маршруты, полученные через OSPF.
· Зачем: Проверка работы OSPF.

· AllowUsers sshuser – разрешает вход только пользователю sshuser.
· >> – добавляет строки в конец файла.
· Зачем: Повышение безопасности SSH.

```bash
systemctl restart sshd
ss -ltnp | grep sshd
```

· ss -ltnp – показывает TCP-сокеты с процессами.
· Зачем: Проверка, что SSH слушает на порту 2026.

```bash

```

· Зачем: Локальная проверка подключения по SSH.

6. Настройка Bind9

```bash

```

· rndc – утилита управления Bind9.
· -a – автоматическая генерация ключа.
· -c – указание файла ключа.
· Зачем: Создание ключа для безопасного управления DNS-сервером.

```bash
cat <<'EOF' > /etc/bind/options.conf
...
EOF
```

· listen-on { localnets; 127.0.0.1; }; – слушать на локальных интерфейсах.
· forwarders { 77.88.8.7; 77.88.8.3; }; – DNS Яндекса для пересылки запросов.
· recursion yes – разрешает рекурсивные запросы.
· allow-recursion { any; } – разрешает рекурсию для всех.
· dnssec-validation no – отключает проверку DNSSEC (для упрощения).
· zone "au-team.irpo" – зона прямого просмотра.
· zone "168.192.in-addr.arpa" – зона обратного просмотра для подсети 192.168.0.0/16.
· Зачем: Конфигурация основных параметров DNS-сервера.

```bash
cat <<'EOF' > /etc/bind/zone/au-team.irpo
...
EOF
```

· SOA – Start of Authority, начальная запись зоны.
· NS – запись DNS-сервера для зоны.
· A – записи соответствия имён IPv4-адресам.
· Зачем: Создание зоны прямого просмотра с записями для всех устройств.

```bash
cat <<'EOF' > /etc/bind/zone/168.192.in-addr.arpa
...
EOF
```

· PTR – записи обратного просмотра (IP → имя).
· Зачем: Создание зоны обратного просмотра для разрешения IP-адресов в имена.

```bash
chown :named /etc/bind/zone/au-team.irpo /etc/bind/zone/168.192.in-addr.arpa
```

· chown :named – изменяет группу файлов на named (пользователь Bind9).
· Зачем: Чтобы Bind9 мог читать файлы зон.

```bash
systemctl enable --now bind
```

· Зачем: Запуск и автозагрузка DNS-сервера.

Чек-лист для проверки выполнения всех задач экзамена.

Общие принципы

Каждый блок содержит команды для проверки определённого аспекта настройки:

· dhcpcd -n – проверка DHCP-клиента.
· ss -lnu | grep 67 – проверка работающего DHCP-сервера.
· ping 8.8.8.8 -c3 – проверка выхода в интернет.
· host имя – проверка DNS.
· ssh -p 2026 – проверка SSH на нестандартном порту.
· ip -d l sh type vlan – проверка VLAN-интерфейсов.
