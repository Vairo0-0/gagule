## Файл 01 [[Образование/Колледж/Мдк/Демо/модуль 2/ISP|ISP]] (Модуль 2-3)

Назначение файла

Настройка ISP как NTP-сервера (Chrony), обратного прокси-сервера (Nginx) и web-аутентификации.

---

1. Настройка Chrony как NTP-сервера

```bash
sed -i 's/pool pool.ntp.org iburst/pool pool.ntp.org iburst prefer minstratum 4/' /etc/chrony.conf
```

· sed -i – редактирует файл на месте.
· s/old/new/ – заменяет строку с пулом NTP-серверов на улучшенную конфигурацию.
· prefer – помечает сервер как предпочтительный.
· minstratum 4 – минимальный слой (стратум) для принятия сервера как источника времени.
· Зачем: Оптимизация клиентской конфигурации Chrony для более точной синхронизации.

```bash
grep pool /etc/chrony.conf
```

· Зачем: Проверка изменений в конфигурации.

```bash
sed -i 's/\#local stratum 10/local stratum 5/' /etc/chrony.conf
```

· local stratum 5 – настройка локального сервера времени с стратумом 5 (работает как резервный источник).
· Зачем: Превращение ISP в локальный NTP-сервер для внутренней сети.

```bash
grep "local stratum" /etc/chrony.conf
```

· Зачем: Проверка включения локального сервера.

```bash
systemctl restart chronyd
```

· Зачем: Применение изменений в конфигурации Chrony.

---

2. Установка и настройка Nginx как обратного прокси

```bash
apt-get update && apt-get install nginx -y
```

· Зачем: Установка веб-сервера Nginx.

```bash
cat << "EOF" > /etc/nginx/sites-available.d/r-proxy.conf
server {
    listen 80;
    server_name web.au-team.irpo;

    location / {
        proxy_pass http://172.16.1.10:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        auth_basic "Restricted Access";
        auth_basic_user_file /etc/nginx/.htpasswd;
    }
}

server {
    listen 80;
    server_name docker.au-team.irpo;

    location / {
        proxy_pass http://172.16.2.10:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF
```

· server_name – доменное имя, для которого работает виртуальный хост.
· proxy_pass – перенаправление запросов на backend-сервер.
· proxy_set_header – передача исходных заголовков клиента на backend.
· auth_basic – включение базовой HTTP-аутентификации.
· Зачем: Создание обратного прокси для двух веб-приложений:
  · web.au-team.irpo → веб-приложение на HQ-SRV (с аутентификацией)
  · docker.au-team.irpo → docker-приложение на BR-SRV (без аутентификации)

```bash
ln -s /etc/nginx/sites-available.d/r-proxy.conf /etc/nginx/sites-enabled.d/
```

· ln -s – создание символической ссылки.
· Зачем: Активация конфигурации сайта (в Nginx принято хранить активные конфигурации в sites-enabled).

```bash
nginx -t
```

· Зачем: Проверка синтаксиса конфигурации Nginx перед запуском.

```bash
systemctl enable --now nginx
```

· Зачем: Запуск и включение автозагрузки Nginx.

```bash
systemctl status nginx
```

· Зачем: Проверка состояния службы Nginx.

---

3. Настройка HTTP Basic Authentication

```bash
apt-get install apache2-htpasswd -y
```

· Зачем: Установка утилиты htpasswd для создания файлов с паролями.

· /etc/nginx/.htpasswd – файл для хранения паролей.
· WEB – имя пользователя для аутентификации.
· Зачем: Создание учётных данных для доступа к защищённому сайту.

```bash

```

· Зачем: Проверка созданного файла с хешами паролей.

```bash

```

· Зачем: Проверка конфигурации и перезапуск Nginx для применения аутентификации.

## Файл 02 [[HQ-RTR и BR-RTR]]

Назначение файла

Настройка статической трансляции портов (Port Forwarding) на маршрутизаторах HQ и BR.

---

1. Настройка Port Forwarding на HQ-RTR

```bash
nft add chain nat prerouting { type nat hook prerouting priority dstnat \; }
```

· nft add chain – добавление новой цепочки правил nftables.
· nat prerouting – имя цепочки.
· type nat hook prerouting priority dstnat – определение цепочки как NAT с перехватом пакетов на этапе prerouting.
· Зачем: Создание цепочки для правил DNAT (Destination NAT).

```bash
nft add rule nat prerouting iif "enp7s1" tcp dport 2026 dnat to 192.168.1.10
```

· iif "enp7s1" – только для пакетов, пришедших на интерфейс enp7s1 (со стороны ISP).
· tcp dport 2026 – для TCP-пакетов с портом назначения 2026.
· dnat to 192.168.1.10 – изменение адреса назначения на 192.168.1.10 (BR-SRV).
· Зачем: Проброс SSH-порта 2026 на BR-SRV.

```bash
nft add rule nat prerouting iif "enp7s1" tcp dport 8080 dnat to 192.168.1.10:80
```

· dnat to 192.168.1.10:80 – изменение адреса и порта назначения.
· Зачем: Проброс порта 8080 на порт 80 BR-SRV (для веб-приложений).

```bash
nft list ruleset
```

· Зачем: Вывод всех текущих правил nftables для проверки.

```bash
nft list ruleset > /etc/nftables/nftables.nft
```

· Зачем: Сохранение текущих правил в конфигурационный файл для постоянного хранения.

```bash
systemctl restart nftables
```

· Зачем: Перезапуск службы для применения новых правил.

---

2. Настройка Port Forwarding на BR-RTR

```bash
nft add chain nat prerouting { type nat hook prerouting priority dstnat \; }
```

· Примечание: Аналогично HQ-RTR.

```bash
nft add rule nat prerouting iif "enp7s1" tcp dport { 8080, 2026 } dnat to 192.168.3.10
```

· { 8080, 2026 } – синтаксис для указания нескольких портов одновременно.
· dnat to 192.168.3.10 – проброс на HQ-SRV.
· Зачем: Проброс портов 8080 и 2026 на HQ-SRV.

```bash
sed -i 's/pool pool.ntp.org iburst/server 172.16.1.1 iburst/' /etc/chrony.conf
```

· server 172.16.1.1 – указание ISP как NTP-сервера.
· Зачем: Настройка синхронизации времени с ISP.

```bash
systemctl restart chronyd && chronyc sources
```

· chronyc sources – вывод источников времени, используемых Chrony.
· Зачем: Проверка синхронизации с NTP-сервером.

---

## Файл 03 [[Образование/Колледж/Мдк/Демо/модуль 2/BR-SRV|BR-SRV]]

Назначение файла

---

```bash
echo "nameserver 192.168.1.10" >> /etc/net/ifaces/enp7s1/resolv.conf
```

2. Установка Samba Domain Controller

```bash
apt-get update && apt-get install -y task-samba-dc
```

· task-samba-dc – метапакет для установки Samba в роли контроллера домена.
· Зачем: Установка всего необходимого для доменных служб.

```bash
rm -f /etc/samba/smb.conf
rm -rf {/var/lib/samba, /var/cache/samba}
```

· Зачем: Удаление старых конфигурационных файлов для чистой установки.

```bash

```

· Зачем: Создание каталога для доменных политик (SYSVOL).

```bash

```

· Интерактивная команда, запрашивает:
  · Realm: AU-TEAM.IRPO (в верхнем регистре)

  · Server Role: dc

  · Administrator password: P@ssw0rd
· Зачем: Создание домена Active Directory.

```bash

cp /var/lib/samba/private/krb5.conf /etc/krb5.conf
```

· Зачем: Замена стандартного файла Kerberos на сгенерированный Samba.

· Зачем: Запуск и проверка служб Samba.

```bash
samba-tool domain info 127.0.0.1
```

· Зачем: Проверка информации о домене.

3. Настройка DNS в Samba


· samba-tool dns add – добавление DNS-записи.
· br-srv.au-team.irpo – DNS-сервер.
· au-team.irpo – доменная зона.
· hq-srv A 192.168.1.10 – A-запись для HQ-SRV.
· -U Administrator – аутентификация под учётной записью администратора домена.
· Зачем: Добавление статических DNS-записей для всех устройств сети.

```bash
samba-tool dns query br-srv.au-team.irpo au-team.irpo @ ALL -U administrator
```

· Зачем: Проверка всех DNS-записей в зоне.

```bash
sed -i 's/nameserver 192.168.1.10/nameserver 127.0.0.1/' /etc/net/ifaces/enp7s1/resolv.conf
```

· Зачем: Настройка сервера использовать свой же DNS (Samba).

## Файл 04 [[HQ-SRV&HQ-CLI]]

Назначение файла

Настройка RAID, NFS-сервера, веб-приложения (LAMP) и NTP-клиента на HQ-SRV и HQ-CLI.

---

1. Настройка NTP-клиента

```bash
sed -i 's/pool pool.ntp.org iburst/server 172.16.1.1 iburst/' /etc/chrony.conf
```

· Примечание: См. объяснение в предыдущих файлах.

```bash
systemctl restart chronyd && chronyc sources
```

· Зачем: Применение настроек и проверка источников времени.

---

2. Создание программного RAID 0

```bash
lsblk
```

· Зачем: Просмотр доступных блочных устройств.

```bash
parted /dev/sdb
```

· Интерактивная утилита для управления разделами:
  ```parted
  mklabel msdos
  mkpart primary 1MiB 100%
  set 1 raid on
  print
  select /dev/sdc
  ```
· mklabel msdos – создание MBR-разметки.
· mkpart primary 1MiB 100% – создание раздела на всё устройство.
· set 1 raid on – установка флага RAID.
· Зачем: Подготовка дисков для создания RAID.

```bash
mdadm --create /dev/md0 --level=0 --raid-devices=2 /dev/sdb1 /dev/sdc1
```

· mdadm --create – создание RAID-массива.
· --level=0 – RAID 0 (объединение без избыточности).
· --raid-devices=2 – количество устройств в RAID.
· Зачем: Создание быстрого дискового массива для NFS.

```bash
mdadm --detail --scan >> /etc/mdadm.conf
```

· Зачем: Сохранение конфигурации RAID для автоматического восстановления при загрузке.

```bash
mkfs.ext4 /dev/md0
```

· Зачем: Создание файловой системы ext4 на RAID-массиве.

```bash
mkdir /raid
```

· Зачем: Создание точки монтирования.

```bash
cp /etc/fstab /etc/fstab.back
echo "/dev/md0 /raid ext4 defaults 0 0" >> /etc/fstab
```

· Зачем: Добавление RAID в fstab для автоматического монтирования при загрузке.

```bash
mount -av
```

· mount -av – монтирование всех файловых систем из fstab с проверкой.
· Зачем: Проверка корректности записей в fstab.

```bash
df -T
```

· Зачем: Просмотр смонтированных файловых систем с их типами.

---

3. Настройка NFS-сервера

```bash
apt-get update && apt-get install nfs-server nfs-utils -y
```

· Зачем: Установка NFS-сервера и утилит.

```bash

```

· Зачем: Создание общего каталога с максимальными правами.


· /raid/nfs – экспортируемый каталог.
· 192.168.2.0/27 – сеть, которой разрешён доступ.
· rw – чтение и запись.
· no_subtree_check – отключение проверки поддеревьев (повышает производительность).
· no_root_squash – доступ root-пользователей клиента с правами root на сервере.
· Зачем: Настройка общего доступа по NFS для клиентов.

```bash

```

· Зачем: Запуск и автозагрузка NFS-сервера.


· Комментарий: Раскомментировать для применения изменений без перезагрузки.
· exportfs -vra – экспорт всех каталогов с подробным выводом.

---

4. Настройка NFS-клиента на HQ-CLI

```bash
mkdir /mnt/nfs
chmod -R 777 /mnt/nfs
```

· Зачем: Создание точки монтирования с полными правами.

```bash
showmount -e hq-srv
```

· showmount -e – показывает экспортируемые каталоги с NFS-сервера.
· Зачем: Проверка доступности NFS-шары.

```bash
cp /etc/fstab /etc/fstab.back
echo "192.168.1.10:/raid/nfs /mnt/nfs nfs rw,soft,_netdev 0 0" >> /etc/fstab
```

· 192.168.1.10:/raid/nfs – сервер и каталог NFS.
· nfs – тип файловой системы.
· soft – мягкое монтирование (при ошибках не зависает).
· _netdev – монтировать после загрузки сети.
· Зачем: Настройка автоматического монтирования NFS при загрузке.


Итоговые рекомендации для Модулей 2-3

Последовательность выполнения:

1. ISP – NTP-сервер и обратный прокси
2. HQ-RTR & BR-RTR – проброс портов
3. BR-SRV – домен, Ansible, Docker
4. HQ-SRV – RAID, NFS, LAMP
5. HQ-CLI – NFS-клиент, тестирование

Критические проверки:

1. NTP: chronyc sources на всех устройствах
2. Проброс портов: nft list ruleset на маршрутизаторах
3. Домен: samba-tool domain info 127.0.0.1
4. Ansible: ansible all -m ping
5. Docker: docker ps и доступность на порту 8080
6. NFS: showmount -e hq-srv и df -T на клиенте
7. LAMP: доступность web.au-team.irpo с аутентификацией

Безопасность:

· HTTP Basic Auth – только базовый уровень защиты
· Samba Domain – используйте сложные пароли
· NFS – ограничьте доступ конкретными подсетями
· SSH – нестандартный порт 2026

Отладка:

· Логи Nginx: /var/log/nginx/
· Логи Samba: /var/log/samba/
· Логи Docker: docker logs <container_name>
· Логи Apache: /var/log/httpd2/

Эта инструкция охватывает все команды из предоставленных файлов с подробными объяснениями назначения каждого параметра и этапа настройки.
